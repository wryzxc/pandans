### 2.2 代码编辑器集成 — CodeMirror 6

#### 什么是 CodeMirror？

CodeMirror 是一个基于 JavaScript 的代码编辑器组件，支持：
- **语法高亮**: 根据语言类型（Python/JS/HTML）对代码着色
- **行号显示**: 左侧显示代码行号，便于定位
- **自动缩进**: 根据语言规则自动调整缩进
- **括号匹配**: 自动补全和匹配括号
- **搜索替换**: 内置查找和替换功能

#### 集成原理

```javascript
// 核心初始化流程
async function initEditor() {
  // 1. 动态加载 CodeMirror 模块
  const { EditorView } = await import('@codemirror/view')
  const { EditorState } = await import('@codemirror/state')
  const { python } = await import('@codemirror/lang-python')
  const { oneDark } = await import('@codemirror/theme-one-dark')
  
  // 2. 创建编辑器状态（文档内容 + 扩展配置）
  const startState = EditorState.create({
    doc: initialCode,           // 初始代码
    extensions: [
      lineNumbers(),            // 行号
      python(),                 // Python 语法支持
      oneDark,                  // 暗色主题
      // ... 其他扩展
    ]
  })
  
  // 3. 实例化编辑器并挂载到 DOM
  cmInstance = new EditorView({
    state: startState,
    parent: document.getElementById('editor')
  })
}
```

#### 与 Vue 的双向绑定

```
Vue ref(code)  ←→  CodeEditor updateListener  ←→  CodeMirror 文档
       │                        │                        │
       │  值变化时               │  文档修改时             │
       └──── 同步更新 ──────────┘                        │
                                                        │
                                  用户输入/粘贴/删除 → 触发
```

### 2.3 Python 浏览器运行环境 — Pyodide

#### 什么是 Pyodide？

Pyodide 是一个将 CPython 解释器编译到 WebAssembly (Wasm) 的项目，使得 Python 代码可以在浏览器中直接运行，无需服务器后端。

#### 工作原理

```
┌───────────────────────────────────────────────┐
│                  浏览器环境                     │
│                                               │
│  Python 源代码                                 │
│       │                                       │
│       ▼                                       │
│  ┌─────────────┐                              │
│  │  Pyodide    │                              │
│  │  (CPython   │                              │
│  │   Wasm)     │                              │
│  └──────┬──────┘                              │
│         │                                     │
│         ▼                                     │
│  ┌─────────────┐    ┌─────────────┐           │
│  │  pandas     │    │  numpy      │           │
│  │  (Wasm)     │    │  (Wasm)     │           │
│  └─────────────┘    └─────────────┘           │
│         │                     │                │
│         ▼                     ▼                │
│    数据处理结果 → stdout → 页面输出显示         │
└───────────────────────────────────────────────┘
```

#### 加载流程

```javascript
// 1. 加载 Pyodide 核心（约 10MB）
const pyodide = await loadPyodide({
  indexURL: 'https://cdn.jsdelivr.net/pyodide/v0.25.1/full/'
})

// 2. 加载额外包（pandas + numpy，约 30MB）
await pyodide.loadPackage('pandas')
await pyodide.loadPackage('numpy')

// 3. 捕获 Python 输出
pyodide.setStdout({
  batched: (msg) => { output += msg }
})

// 4. 执行代码
await pyodide.runPythonAsync(code)
```

#### 限制与注意事项

| 限制项 | 说明 | 应对方案 |
|-------|------|---------|
| 加载时间 | 首次加载需下载 ~40MB 文件 | 预加载 + 缓存到 IndexedDB |
| 包支持 | 并非所有 pip 包都可用 | 使用 Pyodide 官方已编译包 |
| 文件系统 | 浏览器中无真实文件系统 | 使用 Pyodide 虚拟文件系统 |
| 网络请求 | 受浏览器 CORS 限制 | 使用跨域 API 或代理 |

## 3. 概念解析

### 3.1 Pandas 核心概念

#### DataFrame — 数据框架

**定义**: DataFrame 是 Pandas 的核心数据结构，类似于 Excel 中的电子表格或 SQL 中的数据库表。它是一个二维的、大小可变的、异构的表格型数据结构。

```python
# 创建一个 DataFrame
import pandas as pd

data = {
    '姓名': ['张三', '李四', '王五'],
    '年龄': [25, 30, 35],
    '城市': ['北京', '上海', '深圳']
}
df = pd.DataFrame(data)

# 输出:
#     姓名  年龄  城市
# 0   张三  25   北京
# 1   李四  30   上海
# 2   王五  35   深圳
```

**关键属性**:
- **列 (Column)**: 每一列有列名，且数据类型一致
- **行 (Row/Index)**: 每一行有行索引（默认从 0 开始）
- **单元格 (Cell)**: 行列交叉点的具体值

#### Series — 序列

**定义**: Series 是一维带标签的数组，可以理解为 DataFrame 的某一列。

```python
s = pd.Series([1, 2, 3, 4], name='数量')
# 0    1
# 1    2
# 2    3
# 3    4
# Name: 数量, dtype: int64
```

### 3.3 机器学习基础

#### 监督学习 vs 无监督学习

```
                    机器学习
                      │
          ┌───────────┼───────────┐
          ▼           ▼           ▼
    监督学习      无监督学习      强化学习
                      │
              平台涉及的项目
                      │
          ┌───────────┼───────────┐
          ▼           ▼           ▼
     K-Means       RFM        关联分析
     (聚类)       (分箱)      (规则挖掘)
```

| 类型 | 定义 | 数据要求 | 平台项目 |
|------|------|---------|---------|
| **监督学习** | 有标签（已知答案），训练模型预测新数据 | 特征X + 标签Y | — |
| **无监督学习** | 无标签，发现数据内在结构 | 仅特征X | 项目5: K-Means 聚类 |
| **规则学习** | 发现数据中的关联规则 | 事务数据 | 项目3: Apriori |

#### K-Means 聚类原理

```
步骤 1: 随机选择 K 个点作为初始中心
步骤 2: 计算每个点到各中心的距离，归入最近的簇
步骤 3: 重新计算每个簇的中心（取均值）
步骤 4: 重复步骤 2-3，直到中心不再变化

可视化:
  ● 中心点    ○ 数据点

  初始:    ●A      ●B
         ○  ○  ○   ○ ○
        
  迭代1:   ●A      ●B
         (○○○)   (○○)
        
  收敛:    ●A      ●B
         (○○○)   (○○)  ← 中心不再变化
```

### 3.4 数据清洗术语

| 术语 | 英文 | 含义 | 示例 |
|------|------|------|------|
| **缺失值** | Missing Value / NaN | 数据中不存在或未知的值 | 订单金额为空、客户ID为NULL |
| **脏数据** | Dirty Data | 格式错误、单位混乱、含有杂质字符 | "$1,200.00"、"€850" |
| **异常值** | Outlier | 明显偏离正常范围的数据点 | 正常价格100元，突然出现100000元 |
| **重复值** | Duplicate | 完全或部分重复的记录 | 同一订单ID出现两次 |
| **数据类型转换** | Type Casting | 将数据从一种类型转为另一种 | 字符串 "2024-01-01" → datetime |
| **标准化** | Normalization | 将数据缩放到统一范围 | Z-Score: (x - mean) / std |

### 3.5 统计与数学概念

#### 支持度 (Support) 与 置信度 (Confidence)

**场景**: 超市购物篮分析 — "买了尿布的人是否也会买啤酒？"

```
定义:
  Support(A→B) = P(A∩B) = 同时买A和B的订单数 / 总订单数
  Confidence(A→B) = P(B|A) = 同时买A和B的订单数 / 买A的订单数

示例数据 (1000个订单):
  买尿布的订单: 200个
  买啤酒的订单: 300个
  同时买尿布和啤酒: 100个

计算:
  Support(尿布→啤酒) = 100 / 1000 = 10%
  Confidence(尿布→啤酒) = 100 / 200 = 50%

解读:
  10%的订单同时购买了尿布和啤酒
  买了尿布的顾客中，有50%也买了啤酒
```

#### RFM 模型

| 维度 | 全称 | 中文 | 计算方式 | 业务含义 |
|------|------|------|---------|---------|
| **R** | Recency | 最近一次消费 | 今天 - 最后一次消费日期 | 多久没来了？越小越好 |
| **F** | Frequency | 消费频率 | 总订单数 | 来了多少次？越大越好 |
| **M** | Monetary | 消费金额 | 总消费金额 | 花了多少钱？越大越好 |

#### 帕累托法则 (二八法则)

**原理**: 80%的结果来自20%的原因。

在商品分析中的体现:
- 80% 的销售额来自 20% 的商品
- 这 20% 的商品称为"爆款"
- 中间部分称为"腰部商品"
- 剩余大量低贡献商品称为"长尾"

```
销售额累计占比
  │
100%┤                              ●●●● 长尾商品 (大量低贡献)
    │                          ●●
 80%┤                      ●●● ← 分界点: 20%商品贡献80%销售额
    │                  ●●●
    │              ●●●
 60%┤          ●●
    │      ●●● ← 腰部商品
    │  ●●
 20%┤●● ← 爆款商品 (少量高贡献)
    │
  0%└──────────────────────────────────→ 商品排名(按销售额降序)
     0%        20%         50%        100%
```

#### 滚动窗口 (Rolling Window)

**概念**: 不是一次性计算全部，而是在一个滑动的时间窗口内逐步计算。

```
原始数据 (每小时GMV):
  [100, 120, 110, 130, 150, 200, 180, 140, 120, 110, ...]

24小时滚动窗口:
  
  窗口1 [---24个值---] → 计算均值M1、标准差S1
              窗口2 [---24个值---] → 计算均值M2、标准差S2
                        窗口3 [---24个值---] → 计算均值M3、标准差S3

异常判定: 实际值 < M - 2*S → "断崖" (暴跌)
          实际值 > M + 2*S → "突刺" (暴涨)
```
