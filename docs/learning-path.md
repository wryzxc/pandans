<a name="3-每个阶段的学习建议"></a>
## 3. 每个阶段的学习建议

### 📌 学习方法论

#### "三步学习法"

```
第一步: 读代码
    阅读项目初始代码，理解每一行的作用
    ↓
第二步: 改代码
    修改参数、数据，观察结果变化
    ↓
第三步: 写代码
    尝试不看参考答案，独立完成项目
```

#### 每个项目的学习步骤

```
1. 阅读项目需求文档 (5 分钟)
   → 理解业务场景和分析目标

2. 阅读初始代码，逐行注释 (15 分钟)
   → 在代码旁边写注释，解释每行的作用

3. 运行代码，观察输出 (5 分钟)
   → 理解代码的输入和输出

4. 修改代码参数，观察变化 (15 分钟)
   → 改变数据、参数，验证理解

5. 尝试不看代码，自己写 (30-60 分钟)
   → 清空编辑器，凭记忆和理解重写

6. 对照参考答案，查漏补缺 (10 分钟)
   → 对比差异，理解更优写法
```

### 📌 代码阅读技巧

#### 分层阅读法

```
第一层: 整体结构 (5 分钟)
    看 import、变量定义、函数划分
    了解"用了什么库"、"做了什么"

第二层: 核心逻辑 (15 分钟)
    关注 groupby、merge、pivot 等关键操作
    理解"怎么做"

第三层: 细节处理 (15 分钟)
    关注缺失值处理、异常值处理、边界条件
    理解"为什么这么写"
```

### 📌 调试技巧

```python
# 技巧1: 分步打印，观察中间结果

print("Step 1 - 原始数据:")

# 技巧3: 使用 sample 随机查看数据
```

## 4. 进阶方向

完成 10 个项目后，你有以下进阶方向可选：

### 方向 A: 数据可视化

```
Pandas + Matplotlib → Seaborn → Plotly
    ↓
 静态图表          统计图表      交互图表
    ↓                ↓             ↓
 折线图/柱状图    热力图/箱线图   可缩放图表
```

**推荐学习**:
- `df.plot()` — Pandas 内置绘图
- `seaborn.heatmap()` — 热力图（适合相关矩阵）
- `plotly.express` — 交互式图表

### 方向 B: 机器学习

```
数据预处理 (Pandas) → 特征工程 → 模型训练 → 模型评估
    ↓                    ↓           ↓          ↓
 清洗/转换/聚合      编码/缩放/选择   sklearn    准确率/召回率
```

**推荐学习路线**:
1. `sklearn.preprocessing` — 标准化、编码
2. `sklearn.model_selection` — 训练/测试集划分
3. `sklearn.ensemble` — 随机森林、梯度提升
4. `sklearn.metrics` — 评估指标

### 方向 C: SQL 与数据库

```
Pandas (内存计算)    SQL (数据库查询)
    ↓                    ↓
 适合小数据            适合大数据
 灵活操作              高效查询
    │                    │
    └──── 结合使用 ──────┘
         pandas.read_sql()
```

**推荐学习**:
- SQL 基础: SELECT, WHERE, GROUP BY, JOIN
- `pd.read_sql()` — 直接从数据库读取
- `df.to_sql()` — 将 DataFrame 写入数据库

### 方向 D: 大数据工具

```
Pandas (单机)  →  PySpark (分布式)  →  Dask (并行)
   1-10 GB         100 GB+            10-100 GB
   内存加载        分布式计算         分布式Pandas
```

<a name="5-常见问题-faq"></a>
## 5. 常见问题 FAQ

### Q1: 为什么我的代码运行很慢？

**可能原因**:
1. 使用了 `for` 循环逐行处理（应优先使用向量化操作）
2. 数据量过大（考虑分批处理或采样）
3. Pyodide 首次加载需要时间（这是正常的）

**优化建议**:
```python
# ❌ 慢: 逐行循环
for idx, row in df.iterrows():
    df.at[idx, 'new_col'] = row['col1'] * row['col2']

# ✅ 快: 向量化操作
df['new_col'] = df['col1'] * df['col2']
```

### Q2: 如何处理数据中的 NaN？

```python
# 策略1: 删除
df.dropna()             # 删除含 NaN 的行
df.dropna(subset=['a']) # 删除指定列含 NaN 的行

# 策略2: 填充
df.fillna(0)            # 填充为 0
df.fillna(df.mean())    # 填充为均值
df.fillna(method='ffill') # 用前一个值填充

# 策略3: 插值
df.interpolate()        # 线性插值
```

### Q3: groupby 之后如何恢复为 DataFrame？

```python
# groupby 结果默认是 Series 或 DataFrame
# 如果需要重置索引:
result = df.groupby('category')['amount'].sum().reset_index()

# 如果需要命名聚合:
result = df.groupby('category').agg(
    total=('amount', 'sum'),
    count=('amount', 'count')
).reset_index()
```

### Q4: 如何高效地合并两个 DataFrame？

```python
# 内连接: 只保留两边都有的
pd.merge(df1, df2, on='key', how='inner')

# 左连接: 保留左表全部
pd.merge(df1, df2, on='key', how='left')

# 外连接: 保留所有
pd.merge(df1, df2, on='key', how='outer')

# 时间匹配: 取最近的时间记录
pd.merge_asof(df1, df2, on='time')
```

### Q5: Pyodide 加载失败怎么办？

1. 检查网络连接（需要从 CDN 下载 ~40MB）
2. 使用 Chrome/Edge（对 WebAssembly 支持最好）
3. 清除浏览器缓存后重试
4. 如果仍失败，可尝试离线部署 Pyodide

### Q6: 学习 Pandas 需要数学基础吗？

**不需要高深数学**，但以下概念有助于理解：
- **平均值/中位数**: 理解 `mean()`, `median()`
- **标准差**: 理解异常检测中的 2σ 原则
- **百分比**: 理解累计占比、帕累托分析
- **距离公式**: 理解 K-Means 聚类
